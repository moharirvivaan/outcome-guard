Exposes: `src/app/page.tsx` (renders `<DemoApp/>`), client components `DemoApp`, `OutcomeLedger`, `ScoreGauge`, the `useAuditStream` hook (consumes POST /api/audit NDJSON), `outcomePresentation.ts` (classification→color/label/penalty), and `mockAudit` (contract-validated `AuditResult` for NCT01951625, used only by the offline demo).
2. `<OutcomeLedger>` has two modes: TIMER (offline) `audit={AuditResult} intervalMs? onReveal? onComplete?` reveals rows on a timer; STREAMED (live) `audit={shim} streamed matches={OutcomeMatch[]} streaming={boolean}` renders parent-supplied rows as they arrive. `<ScoreGauge score={0-100} caption?/>` tweens toward `score`.
3. DemoApp default = LIVE: "Load demo trial" / "Audit" call POST /api/audit and stream events (registry→paper→match rows→scored→done); demo trial uses the server-side paper fixture, other NCT ids go via Europe PMC. An "Offline demo" checkbox instead renders `mockAudit` with the timer ledger (works with no key/network). Failures (bad id, no publication, missing key, registry error) show a dismissible ErrorBanner; the "reviewer aid, not a verdict" note + per-row confidence are always visible.
