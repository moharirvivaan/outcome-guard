Exposes from `@/lib/engine/matchOutcomes` and `@/lib/engine/extractReported`: `extractReported(paper)` -> `ReportedOutcome[]`, `matchOutcomes(registered, reported)` -> `OutcomeMatch[]`, and convenience `runEngine(registered, paper)` -> `{ reported, matches }`; `paper` may be a string or the fixture's `{ sections: {section,text}[] }`.
Flow: derive `RegisteredOutcome[]` from the registry (primary -> type "primary", registry "otherOutcomes" -> type "secondary"), then call `runEngine(registered, paper)` (or `extractReported` then `matchOutcomes`); `registeredRef`/`reportedRef` on each match index the input arrays, one side null for silently_dropped/silently_added.
Needs `ANTHROPIC_API_KEY` (Sonnet for extract, Opus for match); run the fixture check with `npx tsx src/lib/engine/checkFixture.ts` (exits 0 offline via a committed mock). The provider is pinned to `@ai-sdk/anthropic@^3` (spec v3) to match `ai@6`, so the model passes to `generateObject` natively with no casts — the live extract→match path runs once a key is set.
