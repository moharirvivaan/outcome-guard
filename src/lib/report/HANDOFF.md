Exposes: `computeIntegrityScore(audit): number` and `scoreBreakdown(audit): ScoreBreakdown` (from `@/lib/report/score`); `generateLetter(audit, opts?): Promise<string>` (from `@/lib/report/letter`).
Call: `computeIntegrityScore(auditResult)` -> 0–100 (saturating per-category penalties, not linear); `scoreBreakdown(auditResult)` -> {score, baseline, totalPenalty, terms[3: dropped_primary|dropped_secondary|added, each {count, weight, k, points}], counts (raw + foldedDroppedSecondary/foldedAdded)} for explaining the number; `await generateLetter(auditResult, { model?: "claude-opus-4-8" })` -> COMPare-style letter string. Unit tests: `npx tsx --test src/lib/report/score.test.ts`.
Env: `ANTHROPIC_API_KEY` is OPTIONAL — when unset (or on any AI-SDK error) `generateLetter` falls back to a deterministic, demo-quality template letter, so it always works offline; `score.ts` is pure/deterministic with no env or I/O.
